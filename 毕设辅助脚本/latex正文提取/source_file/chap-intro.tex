 回顾教育领域近几年的发展，出现了越来越多的大规模在线教育平台，提供形式多样的教育资源，如~Coursera~、Khan~Academy~、 网易云课堂等。这些教育资源有两个主要的特点，非结构化、数据量大\cite{lucene1}。 搜索引擎发展至今，主要是基于用户提供的文本信息来进行搜索。而考虑检索教育资源，这些在线教育平台包含了许多非结构化数据，比如图片、视频等资源。如何基于用户提供的文本检索词在这些非格式化数据中快速准确地定位出人们想要的内容，目前依然是一个值得研究的方向。

全文检索是为了有效地存储和搜索大量非结构化数据而诞生的，它主要针对文本数据，并且搜索资源内容而不是外部特征\cite{lucene2}。它提供了一种高效的数据管理工具和强大的查询方法，可帮助人们快速，轻松地找到所需的内容。Lucene~是一个比较受欢迎的全文搜索引擎工具包，广泛应用于许多知名的搜索项目，如~iTunes~、Eclipse~和~Outlook~搜索插件。

全文检索能够满足资源检索的基本需求，用户进行搜索时，可以获得数量充足的查询结果。随着信息量的增长必然会带来巨大的重复及冗余，使得用户进行搜索时，经常会得到许多内容相同、却排行相近的网页结果。与此同时，互联网规模的扩大让用户输入的关键词汇具有了多义性，全文检索并不能处理由自然语言中的“同义词”和“多义词”导致了较低的检索精度。当今用户的需求已经不是获取更多的资源,而是转变为想要获取更加精确、有质量的搜索结果。因此，本文研究了基于~Lucene~和超链分析结合的检索实现，搭建了中文搜索系统，利用排序标准数据集，对排序算法从准确率和召回率方面进行评测分析。

\subsection{研究目的及意义}

面对当今用户需求的变更，研究人员们意识到搜索引擎给用户带来的好处与难处。由于搜索结果太多人们往往需要自行进行检验筛选，导致对搜索引擎的准确性呼声越来越高。这意味着需要提供给用户排序更加优化的搜索结果，提供一个更加高效益的搜索引擎，在满足用户对资源需求的前提下，减少资源的重复性和冗余度，避免不必要资源的出现，从而使得用户可以花费更少时间获取更符合自己预期的资源结果。

国内国外的搜索引擎目前依旧存在着差距，就比如百度，虽然在部分中文查询的情景中可能会优于谷歌，但从搜索结果来看，经常出现具有相同内容的搜索结果会排在相邻的位置中，而谷歌却很少有这种情况。由于商业性质、安全考虑等许多因素，许多公司并不公开其搜索技术及相关研究资源，因此国内搜索领域的技术发展十分受限。本文基于开源搜索平台进行相关研究，采用的也是公开的开源搜索测试数据，对搜索领域的开源及技术发展有促进意义。

在网页排序算法这一方向上，许多研究人员进行了研究与实验，并取得了良好的进展。但互联网本身依旧处于不断变化的阶段，使得网页排序问题变得越来越困难和复杂，仍然需要研究人员投入去攻克互联网变化衍生出来的排序难题。本文在对现有搜索排序算法学习的基础上，研究并实现能够符合用户需求的排序算法，使得用户在进行搜索时可以避免冗余、重复、歧义等资源结果出现，减少用户在获取自己满意的资料时所需要耗费的时间和人力成本。

随着信息化时代的不断发展，面对用户搜索出来的巨大信息量，如果能够恰当地、自动地筛选符合需求的搜索结果，将会极大地提高人们的生产生活效率。

    搜索引擎大致可以分为~3~个工作过程，依次是抓取信息阶段、预处理阶段还有查询阶段。

为了解决用户对搜索结果的需求，有学者尝试在抓取信息阶段上进行改变。设计了一种新的搜索引擎，元搜索引擎\cite{meta_search_engine}。与全文搜索引擎不同,这种搜索引擎依赖其他引擎来进行抓取、存储、索引网页等步骤，即它代替用户将所查询的关键词进行转交操作，让多个其他的搜索引擎对该查询词进行搜索，之后这种搜索引擎结合各个引擎的搜索结果，通过一定的计算选取出各个搜索引擎的优质网页返回给用户，因此可以说元搜索引擎是建立在全文搜索引擎的基础之上。

也有学者研究爬行策略，在抓取信息阶段中采用主题爬虫，即基于主题改变爬虫的具体策略，安排爬虫对网页的抓取顺序\cite{topic_crawl}。在主题爬虫抓取完一个网页之后，需要对待抓取~URL~队列的重要性进行计算后再决定下一步动作。然而，通过主题爬虫获取到的网页资源虽然含有需要的主体信息，但也难以避免混杂进网页噪声，依旧需要投入精力去研究能够高效准确提取网页主题信息的相关技术。

在预处理阶段，研究者们注意到因特网上的超链接结构是个有价值的研究方向，由于网页本身具有的链接属性，其网页自身的重要性取决于有多少数量的其他网页链接到该网页。如果能够利用链接结构特点，就可以简单化复杂的网络结构，进而简化网页排序模型，提高对网页排序结果计算的精确度，从而提供给用户高质量的网络检索信息。

在基于超链接结构进行排序这个方向上有许多研究者提出相关算法，其中比较经典的两个当属~PageRank~算法以及~HITS~算法。然而传统经典的算法也存在缺陷与不足，学者们尝试优化和改进网页排序算法来满足用户的新需求，比如有学者提出基于用户习惯的主题敏感排序算法\cite{user_habit_sense}，可根据不同用户的不同习惯和个性返回相对应的搜索结果，提高用户对搜索引擎的满意程度和引擎的搜索质量，不过这需要先进行用户习惯的采集工作。还有就是针对传统经典算法的改进，比如针对~HITS~算法中存在的主题漂移问题，有学者进行了相关研究，结合文本内容的相似性和链接分析\cite{hits_improve_1}以及采用最大流量算法结合边缘容量\cite{hits_improve_2}等来限制主题漂移的发生。

另外，考虑到关键词存在的二义性，部分研究者基于搜索引擎框架研究分词算法，通过提高分词系统的精确度来避免歧义搜索结果\cite{search_split_word}。

至于在查询阶段，有研究人员提出各种新型的搜索服务。考虑到传统的全文搜索引擎为了兼具简单性和广博性，只能提供给用户一个初始的搜索服务，需要用户自行检验搜索结果，并不能很好的满足个性化需求。对于同样的关键词，所有用户得到的搜索结果是一致的。故有学者提出个性化的搜索引擎\cite{custom_search_engine}，利用信息采集系统收集完用户信息后进行个性化提取，再结合搜索引擎提供个性化信息搜索服务。

与此同时，针对不同的用户群体也出现了不同的搜索引擎。比如针对企业用户，随着企业规模的发展壮大，企业内部资料库也在不断扩充，很多企业开始搭建自己的搜索平台解决定位资料信息的问题。然而如果采用传统数据库检索，不仅会影响查询效率，而且只能处理结构化的数据。尽管如此，试图还原~Google~等搜索引擎的检索效率难度又很高。于是有学者投入到分布式全文检索技术\cite{distribute_search}的研究中去寻找解决方案。

除此之外，由于企业往往是专注于某些特定领域的，于是面向特定领域的垂直搜索引擎\cite{vertical_search}便产生了，致力于弥补通用搜索引擎在专业信息中查询不准确、深度不够等局限性，满足企业内用户专业且深度的搜索需求。垂直搜索引擎需要对特定领域的信息进行专业化的处理，相对来说只需处理较少的信息，所以能够满足用户对于精确性和即时性的需求。而传统的通用搜索引擎，由于需要处理更多数量级的信息，会对特定领域的内容有所忽视。

垂直搜索引擎所具有的优缺点是显而易见的，比如更加专业化、准确、有深度。然而这也是其不足所在，用户需要进行专业化的搜索操作才能找到自己满意的搜索结果，不如传统搜索引擎那么简单方便。由于需要对特定领域进行专业信息处理，需要投入人力成本研究特定领域内的专业特征以及处理过程，难以有通用手段能对不同领域都进行深入的数据挖掘，因为如果没有对特定行业进行特定的分析和研究，同时没有与特定行业从业人员进行相关的交流调查，就无法保证搜索引擎面向特定行业的需求定位是准确的。

\section{研究内容及论文结构}

\subsection{主要研究内容及工作流程}

本文的主要目标是实现经典的网页排序算法――~HITS（Hyperlink - Included Topic Search）~算法，并对其进行评测分析。

为了达到这个目标，按照如图~\ref{fig:workflow}~所示进行工作安排。首先需要建立起一个搜索平台，这里决定基于~Lucene~开源搜索引擎实现。~Lucene~能够提供建立索引、终端交互、相关度排序等搜索相关的功能，在此基础之上再搭建一个搜索平台。

另外，建立搜索平台就需要数据源，即互联网语料库。有多种获取方法，其中一种是自己爬取网页，需要全网不同域名下足够数量的语料库，但考虑到个人能力和时间都有限，所以就采用现有的网上的语料库作为测试数据。这里采用的是搜狗数据源中的全网新闻数据，统计结果表明这其中包含有约~130~万个网页。由于搜狗数据源中含有不规范的数据格式，比如数字、字母都是用全角字符表示，需要先对数据源进行序列化处理后才可以开始建立索引、分词等步骤。

同时，考虑到演示的方便，扩展终端交互实现~Web~界面，用户可以方便地进行相关的搜索并查阅搜索结果，也为后续进行评测分析提供了接口。

这样一个具有~130~万网页的搜索平台就搭建起来了，借助搜索引擎已经能够实现快速搜索，且能够根据相关度排序搜索结果。但考虑到~Lucene~自带的几种分词器对中文分词的支持都较差，而本文的主要目标是进行网页排序的相关工作，因此需要尽可能地避免由于分词效果对本文研究排序算法带来不必要的误差与干扰。需要嵌入较好的中文分词器比如~paoding~分词、~IK~分词等。

接下来就可以开始实现网页排序算法了。本文参考权威论文以及~WiKi~给出的排序伪码，结合自己对排序算法的理解进行主要的代码实现~HITS~算法以及用于评测分析对比的~PageRank~算法。由于排序算法需要依赖链接关系数据库，于是还需要进行建立数据库，数据库交互等步骤。

进入评测分析阶段后，本文依据~MRR（Mean~Reciprocal~Rank）~以及查准率(Precision)作为评测指标进行评测分析。

\subsection{论文结构安排}

第一章：介绍了研究背景，包括研究意义、国内外研究现状及发展趋势等，另外也介绍了本文的主要研究内容。

第二章：对本文搜索平台搭建，排序算法实现所涉及到的相关技术进行简要介绍，包括~Lucene~搜索引擎、分词、排序算法等。

第三章：具体讲解中文搜索平台及排序算法的研究与实现。包括该平台各个模块之间的组织与联系，排序算法的核心思想与实现。

第四章：对实现的排序算法进行评测分析。介绍了评测的相关指标，采用合适的指标进行评测后的结果分析。

第五章：对有关搜索引擎以及网页排序的发展及趋势进行总结与展望。
